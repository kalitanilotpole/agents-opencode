Contributing to agents-opencode

How to contribute
- Fork the repository on GitHub and clone your fork locally.
- Create a feature branch. Naming: feat/brief-description, fix/brief-description, or chore/brief-description.
- Make small, well-tested changes. Prefer small PRs.
- Add or update tests if applicable.
- Run the project's tests locally if available and ensure they pass.
- Submit a pull request against the main branch. Include a concise title and a short description of the changes and why they were made.
- Engage in code review thoughtfully and address feedback promptly.

Git hooks and style
- Follow the project's existing coding style and conventions.
- If there are linters or tests, ensure they pass before requesting a review.

Other notes
- If this repository doesn't have tests or CI yet, explain how you'd test changes locally.
- For questions, open an issue or ask in the project discussion channel.
